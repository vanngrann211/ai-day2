The Data

Meet CIFAR-10

CIFAR-10 is a famous dataset: 60,000 small color photos (32 by 32 pixels), each labeled with one of 10 categories. It is built into TensorFlow, so there are no files to download or upload.

  • 50,000 images for training (teaching the model)
  • 10,000 images for testing (checking if it learned)
  • Every image is already labeled with the correct answer
  • This is exactly the labeled training data we discussed on Day 1
Concept

How a Computer Sees an Image

A computer does not see a picture. It sees numbers. Every image is a grid of pixels, and every pixel is three numbers: how much red, green, and blue it contains.

The Analogy

A 32 by 32 color image is just 32 x 32 x 3 = 3,072 numbers. The neural network learns which patterns of numbers mean "cat" and which mean "truck." That is all image recognition really is: finding patterns in numbers.

CONNECTING TO DAY 2

Remember lists? An image is a list of pixel values. The dataset is a list of images. AI is built on the lists you already understand.

Activity 1

Load the Data

New cell. This loads all 60,000 images in one line:

import tensorflow as tf from tensorflow.keras import layers, models import numpy as np import matplotlib.pyplot as plt (x_train, y_train), (x_test, y_test) = tf.keras.datasets.cifar10.load_data() class_names = ['airplane', 'automobile', 'bird', 'cat', 'deer', 'dog', 'frog', 'horse', 'ship', 'truck'] print("Training images:", x_train.shape) print("Test images:", x_test.shape)
Training images: (50000, 32, 32, 3) Test images: (10000, 32, 32, 3)
Understanding

Reading Those Numbers

The shape (50000, 32, 32, 3) tells you everything about the data:

  • 50000 - the number of images
  • 32, 32 - each image is 32 pixels wide and 32 pixels tall
  • 3 - three color channels: red, green, blue
x AND y

x_train holds the images. y_train holds the correct labels (the answers). This naming is a convention in AI: x is the input, y is the answer. You will see it everywhere.

Activity 2

Look at the Images

New cell. See what you are working with. This uses a for loop, exactly like Day 2:

plt.figure(figsize=(10, 4)) for i in range(10): plt.subplot(2, 5, i + 1) plt.imshow(x_train[i]) plt.title(class_names[y_train[i][0]]) plt.axis('off') plt.show()
WHAT YOU SEE

Ten small photos appear, each with its label. They look blurry because they are only 32 pixels. That is intentional: small images train fast and the model still learns to recognize them.

Activity 3

Prepare the Data

New cell. Use a smaller subset so training finishes quickly in class, and scale the pixels:

x_train_small = x_train[:5000] / 255.0 y_train_small = y_train[:5000] x_test_small = x_test[:1000] / 255.0 y_test_small = y_test[:1000] print("Using", x_train_small.shape[0], "training images") print("Using", x_test_small.shape[0], "test images")
Using 5000 training images Using 1000 test images
Concept

Why Divide by 255?

Pixel values range from 0 to 255. Dividing every pixel by 255 squeezes them into a range of 0 to 1. This is called normalization.

The Analogy

Neural networks learn better with small, consistent numbers. It is like the difference between giving someone directions in inches versus miles. Same information, but one scale is much easier to work with. Models train faster and more reliably on 0-to-1 values.

Activity 4

Build the Neural Network

New cell. This is the model itself - the brain you are building:

model = models.Sequential([ layers.Input(shape=(32, 32, 3)), layers.Conv2D(32, (3, 3), activation='relu'), layers.MaxPooling2D((2, 2)), layers.Conv2D(64, (3, 3), activation='relu'), layers.MaxPooling2D((2, 2)), layers.Flatten(), layers.Dense(64, activation='relu'), layers.Dense(10) ]) model.summary()
DO NOT PANIC

This looks like a lot, but it is just a list of layers - using the list syntax from Day 2. Each layer does one job. The next slide explains every line.

Concept

What Each Layer Does

  • Conv2D - scans the image for visual patterns like edges, curves, and textures
  • MaxPooling2D - shrinks the image down, keeping only the important parts
  • Flatten - turns the 2D grid of patterns into a single long list
  • Dense(64) - a layer of 64 decision-making neurons
  • Dense(10) - the final layer: one output per category (10 categories)
The Big Picture

Early layers find simple patterns (edges). Later layers combine them into complex ones (a cat's ear, a truck's wheel). The final layer votes on which of the 10 categories it sees.

Activity 5

Compile the Model

New cell. Before training, you tell the model how to learn:

model.compile( optimizer='adam', loss=tf.keras.losses.SparseCategoricalCrossentropy(from_logits=True), metrics=['accuracy'] ) print("Model compiled and ready to train")
THREE SETTINGS

optimizer is the strategy for improving. loss measures how wrong the model is (the goal is to make this small). metrics is what we want to watch - accuracy, the percent it gets right.

Concept

How Training Actually Works

Training is a loop, the same idea as Day 2's for loops. For each round, the model:

  • Looks at training images and guesses their categories
  • Checks its guesses against the correct labels
  • Measures how wrong it was (the loss)
  • Adjusts its internal settings slightly to do better next round
  • Repeats. Each full pass through the data is called an epoch
The Analogy

Like studying flashcards. Each pass through the deck, you learn from mistakes and improve. More passes (epochs) usually means better accuracy, up to a point.

Activity 6

Train It

New cell. This is the moment. Run it and watch the model learn:

history = model.fit( x_train_small, y_train_small, epochs=10, validation_data=(x_test_small, y_test_small) )
Epoch 1/10 157/157 - accuracy: 0.28 - val_accuracy: 0.41 Epoch 2/10 157/157 - accuracy: 0.45 - val_accuracy: 0.48 ... Epoch 10/10 157/157 - accuracy: 0.74 - val_accuracy: 0.61
THIS TAKES A FEW MINUTES

Watch the accuracy climb with each epoch. That number rising is your model genuinely learning.

Understanding

Reading the Training Output

  • accuracy - how well it does on the training images it studies
  • val_accuracy - how well it does on test images it has never seen
  • val_accuracy is the honest score. It shows real-world performance
  • Both should rise over the epochs as the model learns
WHAT IS A GOOD SCORE?

With 5,000 images and a simple model, around 60 percent on val_accuracy is a solid classroom result. Random guessing across 10 categories would be 10 percent, so 60 percent means real learning happened.

Concept

Overfitting: The Warning Sign

If training accuracy keeps rising but val_accuracy stops or drops, that is overfitting - the model is memorizing the training images instead of learning general patterns.

The Analogy

Like a student who memorizes the practice test answers but fails the real exam because the questions are different. We saw this on Day 1 with Teachable Machine. The fix is more data or a simpler model.

Activity 7

Test the Final Accuracy

New cell. Get one clear score on the test set:

test_loss, test_accuracy = model.evaluate(x_test_small, y_test_small) print("Final test accuracy:", round(test_accuracy * 100, 1), "percent")
Final test accuracy: 61.3 percent
YOU DID THIS

That percentage is your model correctly identifying images it has never seen before. You built and trained a working neural network from scratch.

Activity 8

Make Predictions

New cell. Ask the model about specific images. This uses np.argmax to find the top guess:

predictions = model.predict(x_test_small) for i in range(5): guess = class_names[np.argmax(predictions[i])] actual = class_names[y_test_small[i][0]] print("Image", i, "- guessed:", guess, "| actual:", actual)
Image 0 - guessed: cat | actual: cat Image 1 - guessed: ship | actual: ship Image 2 - guessed: airplane | actual: ship Image 3 - guessed: airplane | actual: airplane Image 4 - guessed: frog | actual: frog
Activity 9

See the Predictions

New cell. Show the images next to the model's guesses:

plt.figure(figsize=(12, 5)) for i in range(10): plt.subplot(2, 5, i + 1) plt.imshow(x_test_small[i]) guess = class_names[np.argmax(predictions[i])] actual = class_names[y_test_small[i][0]] color = 'green' if guess == actual else 'red' plt.title(guess, color=color) plt.axis('off') plt.show()
GREEN AND RED

Green titles are correct guesses, red are mistakes. Seeing where your model fails is one of the most useful things in AI. The mistakes teach you how to improve it.

Troubleshooting

Common Errors and Fixes

NameError: model not defined

You skipped a cell or Colab restarted. Re-run every cell from the top in order.

Training is very slow

The GPU is not on. Runtime menu, Change runtime type, select T4 GPU, then re-run all cells.

Session crashed / out of RAM

Too many images. Lower the subset back to 5000 training and 1000 test.

Accuracy stuck near 10 percent

Something is off with the data prep. Re-run Activity 3 and check you divided by 255.
